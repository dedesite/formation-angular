# Angular 5 ## Composants et templating ----- # Composants Inspirés des WebComponent Testables Découplés de l'application ----- # Fichiers Une classe TypeScript (fichier `.ts`) Un ou plusieurs `.css` Un seul template `.html` ----- # Premier exercice * `git checkout step-1-component` * Étape 1 sur le wiki * Finir le chapitre "Mes premiers composants" ----- # Classe #1 Directive `@Component` Module ES6 Injection de dépendances ----- # Classe #2 Directive `@Input()` Directive `@Output()` Lifecycle Hook `OnInit` ----- # CSS Classes spécifiques au composant Selecteurs `:host` et `:host-context` Plusieurs fichiers possibles ----- # Html Création d'un tag Html Nouvelles propriété de DOM Nouveaux événements ----- ## Communication inter composants ### Sens parent => enfant #### Propriétés `@Input` Dans la classe `.ts` @Input() name: string; Dans le template `.html` <word-img [name]="currentWord"></word-img> ----- ## Communication inter composants ### Sens enfant => parent #### Propriétés `@Output` Déclaration d'événement dans la classe : @Output("founded") isFound: EventEmitter = new EventEmitter(); Dans le template `.html` <word-grapheme (founded)="onFound($event)"> ----- ## Communication inter composants ### Sens enfant => parent #### Variable de référence (`#var`) Uniquement utilisable dans les templates <input #phone placeholder="phone number"> <button (click)="callPhone(phone.value)">Call</button> ----- ## Communication inter composants ### Sens enfant => parent #### Directive `@ViewChild` Dans la classe import { ViewChild } from '@angular/core'; //.. @ViewChild(CountdownTimerComponent) private timerComponent: CountdownTimerComponent; ----- ## Communication inter composants Via un Service partagé Entre parent et enfants ----- # Templating Intégré à html Peu de structure de controle Extension des propriétés DOM très particulière ----- # Template expression Inpterpolation via `{{}}` => escape les tags <h1>{{title}}</h1> Property binding via `""` => enlève les tags <span [hidden]="isUnchanged">changed</span> Local au template et au composant (pas de Math.max) ----- # Conseils Pas de changement d'état Rapidité => beaucoup de rafraichissement possibles Simplicité => fonction si besoin Idempotence => pas de changement de valeur ----- # Template statement Réponse à un évenement A droite d'un binding d'event (click)="onClick()" Local au template et au composant ----- # Data-binding One-way : Données => Vue : `{{expression}}` et `[target]="expression"` One-way : Vue => Données : `(target)="statement"` Two-way : Vue <=> Données : `[(target)]="expression"` ----- # Attribut HTML vs. propriétés DOM Attributs statiques Attributs valeur initiale Propriétés dynamiques Pas de matching 1:1 ----- # Binding targets : propriétés Propriétés d'élément Html : <img [src]="wordImageUrl"> Propriétés de composant : <word [word]="currentWord"></word> Propriétés de directives : <div appDraggable [data]="grapheme"> ----- # Binding targets : event Évenement du dom : <button (click)="onSave()">Save</button> Évenement de composant : <word-grapheme (founded)="onFound()"></word-grapheme> Évenement de directives : <div appDroppable (dropped)="onDrop(grapheme)"> ----- # Autres Binding Sur attributs (l'exception) : <button [attr.aria-label]="help">help</button> Activation conditionnel d'une classe CSS : <div [class.special]="isSpecial">Special</div> Définition d'une propriété de style : <div [style.color]="grapheme.color"> ----- # Attention Ne pas oublier les `()` et `[]` Sauf pour binding statique : <word word="pamplemousse.jpg"></word> Ou en utilisant l'interpolation : <img src="{{imageUrl}}"> ----- # Two way data binding #1 <input [(ngModel)]="child.name"> Sucre syntaxique Un input `[]` et un event `()` ----- # Two way data binging #2 Event doit avoir même nom que propriété + `Change` <app-sizer [(size)]="fontSizePx"> egal <app-sizer [size]="fontSizePx" (sizeChange)="fontSizePx=$event"> ----- # Attribute directives Peuvent être créées `ngClass` et `ngStyle` built-in pour l'activation de plusieurs class css et styles <div [ngClass]="currentClasses"> `ngModel` two-way binding des `input` <input [(ngModel)]="child.name"> ----- # Structural directives #1 ngIf <word *ngIf="currentWord"> ngFor <div *ngFor="let wordGrapheme of word.graphemes"> ----- # Structural directives #2 ngSwitch <div [ngSwitch]="currentHero.emotion"> <happy *ngSwitchCase="'happy'" [hero]="currentHero"></happy> <sad *ngSwitchCase="'sad'" [hero]="currentHero"></sad> <conf *ngSwitchCase="'confused'" [hero]="currentHero"></conf> <unknown *ngSwitchDefault [hero]="currentHero"></unknown> </div> ----- # Structural directives #3 Attention au signe `*` ! <li *ngFor="let contact of contacts"> <contact-card [contact]="contact"></contact-card> </li> devient <ng-template ngFor let-contact [ngForOf]="contacts"> <li> <contact-card [contact]="contact"></contact-card> </li> </ng-template> ----- # Variables de templates Variables locales <div *ngFor="let wordGrapheme of word.graphemes"> Variables "référence" `#var` <input #phone placeholder="phone number"> ----- # Template expression operator Opérateur pipe `|` `{{title | uppercase}}` Opérateur "safe navigation" '?' `Prénom de l'enfant {{child?.name}}` non-null assertion '!' => spécifique
